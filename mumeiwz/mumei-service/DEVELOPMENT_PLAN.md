# 沐美服务 - 功能开发计划

## 📋 总体目标
让个人用户"**快速注册 → 快速付费 → 快速使用**"

---

## 🚀 第一阶段：简化注册（预计 2 小时）

### 目标：用户 30 秒完成注册

#### 1.1 简化注册流程
```
当前流程: 邮箱 → 收验证码 → 输入验证码 → 设置密码 → 注册成功 (5步)
优化后:   邮箱 + 密码 → 直接注册 (1步)
```

**修改文件：**
- `server.js` - 简化注册 API
- `public/panel.html` - 简化注册表单
- `db-sqljs.js` - 移除验证码必填

**验收标准：**
- [ ] 邮箱+密码直接注册成功
- [ ] 注册后自动登录
- [ ] 跳转到控制台

#### 1.2 邮箱验证码改为可选
```
- 简化注册：无需验证码
- 敏感操作时仍需验证（如修改密码、删除账户）
```

**验收标准：**
- [ ] 注册无需验证码
- [ ] 登录无需验证码（可选）
- [ ] 敏感操作需要验证码

#### 1.3 首次注册赠送配额
```
注册即送:
- 50 次免费 API 调用
- 有效期 7 天
- 过期提醒邮件
```

**验收标准：**
- [ ] 新用户注册后显示赠送配额
- [ ] 配额 7 天后过期
- [ ] 过期前发送提醒邮件

## 🚀 第三阶段：支付系统（预计 4 小时）

### 目标：3 步完成付费

#### 3.1 接入微信/支付宝
```
方案：
- 个人可用的支付接口
- 方案1: 易支付 (费率 0.6%-1%)
- 方案2: 码支付 (费率 0.6%-1%)
- 方案3: 自建支付 (需企业资质)
```

**配置项：**
```
.env 文件添加:
PAYMENT_TYPE=epay          # 易支付
EPAY_URL=https://api.xxx.com
EPAY_PID=你的商户ID
EPAY_KEY=你的商户密钥
```

**验收标准：**
- [ ] 扫码支付可用
- [ ] 支付成功自动升级套餐
- [ ] 支付结果通知

#### 3.2 订单管理
```
数据库表:
- orders (订单表)
- 字段: id, user_id, plan_id, amount, status, pay_time
```

**验收标准：**
- [ ] 订单创建成功
- [ ] 订单状态更新
- [ ] 订单查询

#### 3.3 自动续费
```
到期前:
- 7 天前: 邮件提醒
- 3 天前: 邮件提醒
- 1 天前: 邮件提醒
- 到期日: 降级为免费版（保留数据）
```

**验收标准：**
- [ ] 到期提醒邮件
- [ ] 降级逻辑正确
- [ ] 数据保留

## 🚀 第五阶段：营销功能（预计 2 小时）

```
邀请机制:

- 被邀请人付费 → 邀请人获得 ¥5 返利

```

```
- 社区活跃度
```

**验收标准：**
- [ ] 热门工具展示

```

- 每日 100 次 (次日重置)
```
